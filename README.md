# 新好感度 (FavourPro)

LDMBot 好感度、印象与关系管理插件，按群隔离。

## 工作原理

每次 LLM 对话时，插件自动将当前用户的好感状态注入上下文，LLM 在回复中附带状态更新块，插件解析后自动更新数据并从回复中移除标记。

状态块格式：`[Favour: <数值>, Attitude: <印象>, Relationship: <关系>]`

### 核心机制

- **好感度范围**：-100 ~ 100，单次变化上限 ±8（好感度为负时正面最高 +8）
- **数据隔离**：按群 ID 隔离，私聊统一归入"私聊"分区
- **自动备份**：每次保存前自动备份为 `好感数据.json.bak`
- **昵称跟踪**：自动记录用户最新昵称

## 指令

| 指令 | 权限 | 说明 |
|------|------|------|
| `查询好感` / `好感度` / `查看好感` | 管理员可查他人，普通用户查自己 | 查看用户好感状态 |
| `设置好感 <用户ID> <好感值>` | 管理员 | 设置指定用户好感度 |
| `设置印象 <用户ID> <印象>` | 管理员 | 设置指定用户印象 |
| `设置关系 <用户ID> <关系>` | 管理员 | 设置指定用户关系 |
| `设置全部 <用户ID>` | 管理员 | 一次性设置好感/关系/印象/名字（多行格式） |
| `重置好感 <用户ID>` | 管理员 | 重置指定用户为默认状态（保留昵称） |
| `重置负面` | 管理员 | 重置当前群内所有好感度为负的用户 |
| `重置全部` | 管理员 | 清空当前群所有用户数据 |
| `好感排行 [数量]` | 管理员 | 按好感度降序排行 |
| `负好感排行 [数量]` | 管理员 | 按好感度升序排行 |
| `撤销修改` | 管理员 | 从备份恢复，撤销上次修改 |

### 设置全部格式

```
设置全部 <用户ID>
<好感度数值>
<关系描述>
<印象描述>
<名字>（可选）
```

### 示例

```
设置好感 123456 50
设置印象 123456 热心善良
查询好感 @某人
好感排行 5
```

## LLM 工具

| 工具名 | 说明 |
|--------|------|
| `favour_rank` | 查询好感度排行，支持正序/倒序，仅管理员可用 |

## 配置项

| 配置 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 群组隔离 | bool | true | 按群隔离数据（暂不支持全局模式） |
| 好感度提示词 | text | （内置长文） | 注入给 LLM 的好感度系统提示词，清空可恢复默认 |

## 数据结构

数据存储于插件数据目录的 `好感数据.json`：

```json
{
  "群号": {
    "用户ID": {
      "favour": 0,
      "attitude": "中立",
      "relationship": "陌生人",
      "name": "昵称"
    }
  }
}
```

默认状态：好感度 0，印象"中立"，关系"陌生人"。

## 安全机制

- 好感度防注入：用户直接贴出状态描述会被视为不友好互动
- 单次变化限幅：±8 上限，防止极端跳变
- 范围钳制：自动修正到 [-100, 100]
- 数据备份：保存前自动备份，支持一键撤销
